# Examples

What a project's `.vibe/` looks like for each preset — these are the exact
files `install.sh` seeds (rendered from `src/templates/`, regenerated by
`examples/render.sh`, and kept honest by `verify.sh`, which diffs them
against a real install of every preset).

```text
examples/
├── minimal/   # devcontainers/base:debian — shell tools, Claude Code, uv
├── python/    # devcontainers/python:3.14
├── bun/       # + Bun (INSTALL_BUN)
└── roblox/    # python base + Rokit (INSTALL_ROKIT)
```

Each directory holds the two files a project actually edits day-to-day:

- **`compose.yaml`** — the project's container policy, merged on top of the
  harness base (`src/compose/base.yaml`). Build args, ports, extra mounts.
- **`config.env`** — behavior toggles: agent command, tmux, bootstrap
  strictness, required commands, preview/review settings.

A full installed project additionally gets `vibe` (wrapper + root symlink),
`AGENTS.md` (container rules for agents), `project/` lifecycle hooks,
`yazi/` review preferences, and the harness submodule — those are identical
across presets; see `src/templates/`.

**[`extensions/`](extensions/)** holds image-extension examples
(playwright, blender) — project `.vibe/Dockerfile`s chained onto the shared
image per [docs/extending.md](../docs/extending.md).

Worked recipes on top of these bases live in `docs/`:
[roblox](../docs/roblox.md) (loopback Rojo port),
[browser automation](../docs/browser-automation.md) (playwright deps),
[local models](../docs/local-models.md) (host Ollama routing).
